OpenAI Ads API 운영 검토 초안

운영 대시보드, 예산 소진 모니터링, 캠페인·광고 소재 자동화에서 무엇을 API로 처리할 수 있는지 정리한 검토용 별도 페이지입니다.

별도 초안 페이지 · 메인 메뉴 미노출
최종 업데이트: 2026-06-18

운영 관점 결론

바로 설계 가능한 영역

  • 캠페인, 광고그룹, 광고의 생성·조회·수정·상태 변경
  • 이미지 파일 업로드 후 광고 소재에 재사용
  • 계정·캠페인·광고그룹·광고 단위 성과 리포트 조회
  • 노출, 클릭, 비용, CTR, CPC, CPM 기반 운영 대시보드

보수적으로 봐야 할 영역

  • 완전한 실시간 수치 또는 반영 지연 SLA는 문서상 확인 필요
  • 전환 성과는 픽셀·이벤트 설치 상태와 매칭 품질에 의존
  • 상품 피드 광고는 피드 준비·매핑·측정 구조를 별도 점검

API 범위 밖으로 보는 영역

  • 인보이스, VAT, 결제수단, 크리테오 정산 정책
  • 계정 Spend Cap 승인·상향 요청
  • Ads Manager 계정 신규 provisioning 승인
  • 공식 문서에 없는 내부 영업 조건 자동 확정

실시간 노출량·효율·예산 소진을 볼 수 있나?

가능한 방향: OpenAI Ads API의 Insights는 광고 계정, 캠페인, 광고그룹, 광고 범위의 집계 성과 데이터를 조회하며 공식 지표로 impressions, clicks, spend, ctr, cpc, cpm을 제공합니다. 따라서 운영 대시보드에서 노출·클릭·비용·효율 지표를 가져오는 설계는 가능합니다.

단, 공식 문서에서 “완전한 실시간” 갱신 주기나 지연 SLA를 확인하지 못했으므로, 초기 구현은 30분~1시간 단위 주기적 동기화와 “최근 수집시각” 표시를 권장합니다.

노출량 조회 가능 클릭·비용 조회 가능 CTR/CPC/CPM 조회 가능 완전 실시간 SLA 미확인

API 활용 범위 매트릭스

운영 업무 판정 사용 API/근거 운영 메모
캠페인 목록·생성·수정·일시정지 가능 Campaigns API: create, list, retrieve, update, state change 예산, 상태, 기간 같은 운영 메타데이터를 서버에서 관리 가능.
광고그룹 목록·생성·수정·보관 가능 Ad Groups API: create, list, retrieve, update, state change 입찰가, 키워드/컨텍스트 힌트, 기본 URL 자동 세팅 후보.
광고 소재 목록·생성·수정·상태 변경 가능 Ads API: create, list, retrieve, update, state change 제목·설명·랜딩 URL·이미지 파일 ID 기반으로 소재 생성.
이미지/파일 등록 가능 Files API: remote image 또는 binary upload 후 file ID 재사용 광고주가 올린 이미지를 검수 후 OpenAI Ads 파일로 업로드 가능.
노출·클릭·비용·효율 대시보드 가능 Insights API: ad account, campaign, ad group, ad scope의 집계 성과 캠페인별 소진율, 저성과, 미노출 경보를 만들 수 있음.
국가·디바이스·상품 기준 세그먼트 분석 가능 Insights segments: product, country, device 문서상 세그먼트는 1개 추가 차원 기준으로 설계하는 것이 안전.
지역 타겟팅 설계 가능 Campaign Targeting guide: country, region, DMA 초기 한국 운영은 KR 중심, 추후 국가 확장 시 코드 검증 필요.
상품 피드 기반 광고 확인 필요 Product feeds guide 피드 포맷·매핑·상품별 리포팅은 별도 파일/운영 플로우 점검 필요.
인보이스·VAT·결제수단·정산 자동화 문서 미확인 공개 Ads API 문서 범위에서 확인되지 않음 나스미디어/크리테오 정산 운영 프로세스로 유지.
Account Spend Cap 상향·승인 문서 미확인 공개 Ads API 문서 범위에서 확인되지 않음 OpenAI 담당자 요청·승인 루트로 처리하는 것이 안전.

추천 MVP 구현안

1단계: 읽기 전용 운영 대시보드

  • API key를 서버 환경변수로 저장하고 브라우저에는 노출하지 않음
  • 캠페인·광고그룹·광고 목록을 주기적으로 동기화
  • Insights를 시간 단위 또는 일 단위로 수집해 Supabase에 저장
  • 최근 수집시각, 예산 대비 소진율, 노출/클릭/비용 추이를 표시

2단계: 운영 알림과 제한적 쓰기

  • 소진율 80%/95%, 미노출, CPC·CPM 급등 경보
  • 승인된 워크북만 Campaigns/Ad Groups/Ads API로 생성
  • 일시정지·재개 같은 상태 변경은 관리자 확인 후 실행
  • 수정 이력은 별도 ops log에 남겨 추적 가능하게 설계

예시 호출 형태

GET https://api.ads.openai.com/v1/ad_account/insights Authorization: Bearer $OPENAI_ADS_API_KEY fields[]=campaign.id fields[]=campaign.name fields[]=campaign.impressions fields[]=campaign.clicks fields[]=campaign.spend fields[]=campaign.ctr fields[]=campaign.cpc fields[]=campaign.cpm time_granularity=daily aggregation_level=campaign

실제 파라미터와 time range 구성은 공식 Insights 문서를 기준으로 구현합니다. API key는 Ads Manager의 Settings에서 발급하고, 모든 요청은 Authorization: Bearer $OPENAI_ADS_API_KEY 헤더를 사용합니다.

보안·운영 가드레일

  • API 키는 Vercel/GitHub Secrets 또는 서버 환경변수에만 저장합니다.
  • 브라우저 JS에서 Ads API를 직접 호출하지 않습니다.
  • 쓰기 API는 승인된 관리자 액션 또는 승인된 워크북에만 연결합니다.
  • 성과 데이터 저장 시 광고주명, 캠페인명, 비용 데이터 접근 권한을 제한합니다.
  • 공식 문서에 없는 정산·계정 승인 값은 RAG 내부 확정 문서 또는 담당자 확인으로만 안내합니다.
